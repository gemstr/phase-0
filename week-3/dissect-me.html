What is Git?

What is Git? Git is a command line program that is used for version control. This means with Git, you can save multiple versions of your work and go back to various versions/states of files.

Using Git is like owning a bunch of USB thumb drives. The original thumbdrive that the source code/project resides is called 'master'. 'Master' can be manipulated, but it is not advised. It is better practice to 'clone' the master to another thumbdrive so it can be worked on without interrupting the 'master' USB drive. The same concept applies to git 'braching'. After the user is done working, changes can be merged back to the master USB with file comparison. Then it is up to the user to delete the branch (throw away USB Drive) as it isn't needed anymore.

Git helps users keep track of their project's file changes thru the use of commits. Everytime a user "commits" a checkpoint of the file(s) it's state is saved so it can be reverted to a specific commit point in the past. But how does Git know which files to include in it's commit? It all starts with the 'git add' command.

Github is a social website that allows coders to collaborate together using Git by hosting code thru peer review. Because this code is hosted online it can be cloned to a local computer and worked on from there! One of the many benefits of hosting on Github!
